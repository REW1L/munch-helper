# MunchHelper

A mobile application that can create a room for a Munchkin game with players with their characters.

# Problem statements

- Multiple players sitting at a table can’t see what other players have on board
- Complex calculations with multiple cards on board complicate gameplay

# Project goals

The project will improve experience of players introducing a mobile application for board games

# Project objectives

Create a mobile application for managing and tracking a game process for multiple players

# Quality objectives

- Test coverage should be more than 70%
- Integration and system testing should be automated
- Score of the applications in markets should be 4+ after 6 months
- Fixing issues should take less than 2 weeks
- Technical debt shouldn’t exceed a month of work
- Technical objectives
    - iOS and Android are primary targets
    - React Native should be used
    - Cloud native and server less solutions should be preferred
    - Extendibility should be available
- Performance objectives
    - The project should take 1-2 months

# Project scope

- What is included
    - iOS and Android applications
        - Anonymous players
            - Limits on creating games
        - Registering
            - Points for creating games
            - Buying points
        - Munchkin game helper
            - Creating and joining a game
            - Players’ characteristics management
            - A battle management
            - Logs of a game events
    - Backend in a public cloud provider
    - Testing and production environments
    - Code and requirements are covered by tests
    - Continuous Deployment
- What is not included
    - Web version
    - Other games
    - Offline mode

# Project Deliverables

- iOS application
- Android application
- Backend for the application
- Testing and production environments
- Continuous Deployment pipelines
- Documentation of architecture and operation processes

# Success criteria

- iOS application is delivered to the store
- Android application is delivered to the store
- Applications have scores 4+
- Test coverage is 70+%
- Update of the application takes less than 2 weeks
- Application covers its operational costs

# Project assumptions

- Marketing is not needed (doesn’t cost money)
- In-app purchases do not require additional configuration

# Project risks

- Licensing for games is unclear and can lead to renaming

# Project Constraints

- Project operational costs on the start are less than $150 per month
- Project should be delivered in 1-2 months
